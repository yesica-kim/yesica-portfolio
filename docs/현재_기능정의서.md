# Yesica Portfolio 현재 기능정의서

## 현재 버전

v0.1.0

## 기능 범위

현재 버전은 포트폴리오 사이트의 기본 페이지, 라우팅, 데이터 구조, 프로젝트 상세 템플릿, 다크 테마 Hero, 배포 운영 구조를 포함한다. 실제 프로젝트 콘텐츠는 placeholder와 초기 요약 중심으로 구성되어 있다.

## 화면/페이지 구성

- `/`: Home
  - Dark Hero
  - Selected Projects
  - How I Work
  - Short About
  - Contact CTA
- `/about`: About
- `/projects`: Projects 목록
- `/projects/[slug]`: 프로젝트 상세 공통 템플릿
  - `/projects/schedule-block`
  - `/projects/walla`
  - `/projects/wsop-plus`
  - `/projects/pokerstake`
- `/contact`: Contact
- `/writing`: Brunch/Writing placeholder
- `/schedule-block`: Schedule Block 공식 소개 페이지
- `/schedule-block/privacy`: Schedule Block 개인정보처리방침
- `/schedule-block/terms`: Schedule Block 이용약관
- `/schedule-block/support`: Schedule Block 지원

## 핵심 데이터 구조

프로젝트 데이터는 `src/data/projects.ts`에서 관리한다.

주요 필드:

- `slug`
- `title`
- `category`
- `description`
- `role`
- `platform`
- `period`
- `status`
- `accent`
- `focus`
- `overview`
- `problem`
- `goal`
- `screens`

## 주요 기능 상세

### 공통 레이아웃

- `src/app/layout.tsx`에서 전역 레이아웃 구성
- `SiteHeader`, `SiteFooter` 공통 사용
- 기본 언어는 `ko`

### Header

- 좌측 텍스트 로고: `Yesica-Portfolio`
- 중앙 메뉴: About, Projects, Process, Brunch
- 우측 CTA: Contact Me

### Home Hero

- 다크 네이비 배경
- 큰 타이포그래피
- blue-green gradient text
- floating product/system card UI
- trust bar
- 모바일에서는 카드가 세로 스택으로 재배치

### Projects

- 프로젝트 카드 그리드
- 데이터 파일 기반 렌더링
- 각 프로젝트 상세 페이지로 연결

### Project Detail

공통 구조:

- Project Hero
- Overview
- Problem
- Goal
- Process
- Design Solution placeholder
- Result / Learning

### Motion

- Framer Motion 사용
- 페이지/섹션 진입 fade-up
- 카드 hover 이동
- Hero 카드 미세 parallax/hover 느낌

### Schedule Block 공식 페이지

- 기존 Schedule Block 프로젝트의 `website` 정적 페이지 내용을 기준으로 포트폴리오 하위 경로에 이식
- 포트폴리오 공통 Header/Footer 없이 독립 공식 페이지처럼 노출
- 앱 아이콘은 `public/schedule-block/app-icon.png`에서 제공
- 개인정보처리방침, 이용약관, 지원 페이지를 App Store/Google OAuth 검수 대응용 공식 문서 경로로 사용할 수 있음

## 사용자 플로우

1. 사용자가 Home에 진입한다.
2. Hero에서 Yesica의 방향성과 주요 작업 톤을 파악한다.
3. Projects로 이동해 작업 목록을 확인한다.
4. 프로젝트 상세 페이지에서 문제, 목표, 과정, 솔루션 영역을 본다.
5. About 또는 Contact로 이동해 소개와 문의 정보를 확인한다.

## 외부 연동

## 저장/동기화 방식

- 사이트 콘텐츠는 코드와 Markdown/TypeScript 데이터로 관리한다.
- 프로젝트 데이터는 `src/data/projects.ts`에서 관리한다.
- 문서는 `docs/` 하위 Markdown 파일로 관리한다.
- 배포는 GitHub 브랜치와 Vercel 프로젝트 연결을 사용한다.

## 예외/제한사항

- 실제 이메일, 브런치, GitHub, LinkedIn 링크는 placeholder 상태다.
- 프로젝트 상세 콘텐츠는 일부 placeholder 상태다.
- mobimobi는 현재 Hero 카드에는 있으나 주요 프로젝트 데이터에는 포함하지 않는다.
- Writing/Brunch 페이지는 placeholder 상태다.
- Schedule Block 공식 페이지는 기존 정적 페이지 내용 기준이며, 앱 업데이트 시 정책 문구도 함께 갱신해야 한다.
- 디자인 이미지는 실제 이미지가 아니라 gradient/mockup placeholder 중심이다.

## 아직 구현하지 않은 기능

- 실제 프로젝트 이미지/영상 연결
- 프로젝트별 정식 케이스 스터디 작성
- Live Product / App Store / GitHub 링크 필드 추가
- Brunch 글 목록 연결
- Resume 페이지
- Lab 또는 Studio 페이지
- SEO/OG 이미지 정리

## 검증 기준

- `npm run lint` 통과
- `npm run build` 통과
- Home, About, Projects, Project Detail, Contact, Writing 라우트 접근 가능
- Schedule Block 공식 페이지 라우트 접근 가능
- 데스크톱과 모바일에서 Hero 텍스트와 카드가 겹치지 않음
- `main` 브랜치 배포는 실서버에 반영
- `dev` 브랜치 배포는 개발서버에 반영
