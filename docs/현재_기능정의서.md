# Yesica Portfolio 현재 기능정의서

## 현재 버전

v1.0.0-dev

## 기능 범위

현재 버전은 포트폴리오 사이트의 기본 페이지, 라우팅, 데이터 구조, 프로젝트 상세 템플릿, 다크 테마 Hero, 배포 운영 구조를 포함한다. Home은 한국어 중심 기본 화면과 영어 화면을 분리해 운영하며, 실제 공개 화면에 어색한 임시 문구가 노출되지 않도록 1차 포트폴리오용 카피로 정리되어 있다.

## 화면/페이지 구성

- `/`: Home
  - Dark Hero
  - Selected Projects
  - About / Intro
  - Experience
  - Product Thinking
  - Writing / Experiments
  - Contact CTA
- `/en`: English Home
  - Korean Home과 같은 섹션 구조를 영어 중심 카피로 제공
- `/about`: About
- `/projects`: Projects 목록
- `/projects/[slug]`: 프로젝트 상세 공통 템플릿
  - `/projects/schedule-block`
  - `/projects/walla`
  - `/projects/wsop-plus`
  - `/projects/pokerstake`
- `/contact`: Contact
- `/writing`: Brunch/Writing
- `/schedule-block`: Schedule Block 공식 소개 페이지
- `/schedule-block/privacy`: Schedule Block 개인정보처리방침
- `/schedule-block/terms`: Schedule Block 이용약관
- `/schedule-block/support`: Schedule Block 지원

## 주요 기능 상세

### 공통 레이아웃

- `src/app/layout.tsx`에서 전역 레이아웃 구성
- `SiteHeader`, `SiteFooter` 공통 사용
- 기본 언어는 `ko`

### Header

- 좌측 텍스트 로고: `Yesica-Portfolio`
- 중앙 메뉴는 `Works`, `About`, `Experience`, `brunch` 영문 라벨로 노출
- 우측 CTA는 라인 없는 filled gradient `Contact` 버튼으로 노출
- `KO / EN` 언어 전환 버튼 제공

### Home Hero

- 다크 네이비 배경
- Korean Home은 영문 eyebrow와 한국어 중심 2줄 타이포그래피
- English Home은 영어 중심 2줄 타이포그래피
- blue-green gradient text
- CTA는 `View Projects`로 통일
- 정보량을 줄인 floating product/system card UI
- 마우스 움직임에 따라 배경 glow가 은은하게 반응
- Hero 카드 영역은 미세 parallax로 floating 느낌 제공
- 모바일에서는 카드가 세로 스택으로 재배치

### Experience

- 2011–2025 경력 흐름을 타임라인/카드 구조로 제공
- 공공기관/도서관 플랫폼, 브랜드 운영, 관리자 시스템, 글로벌 서비스, 개인 제품 제작 흐름을 요약
- 경력 연수 반복보다 실제 수행 업무와 서비스 운영 맥락을 보여주는 역할
- NSUSLAB 표기 기준으로 글로벌 포커 플랫폼, GGPoker, WSOP+, PokerStake 관련 업무를 설명

### Projects

- 프로젝트 카드 그리드
- 데이터 파일 기반 렌더링
- 각 카드에 기간, 플랫폼, 역할, 상태, 핵심 성과를 함께 노출
- 한국어/영어 Home에서 프로젝트 카드의 주요 문구를 현재 언어에 맞춰 노출
- 각 프로젝트 상세 페이지로 연결

### Project Detail

공통 구조:

- Project Hero
- Overview
- Problem
- Goal
- Process
- Design Solution
- Result / Learning

### Motion

- Framer Motion 사용
- 페이지/섹션 진입 fade-up
- 카드 hover 이동
- Hero 카드 미세 parallax/hover 느낌
- Hero 배경 spotlight glow가 마우스 위치에 따라 부드럽게 이동

### Schedule Block 공식 페이지

- 기존 Schedule Block 프로젝트의 `website` 정적 페이지 내용을 기준으로 포트폴리오 하위 경로에 이식
- 포트폴리오 공통 Header/Footer 없이 독립 공식 페이지처럼 노출
- 앱 아이콘은 `public/schedule-block/app-icon.png`에서 제공
- 개인정보처리방침, 이용약관, 지원 페이지를 App Store/Google OAuth 검수 대응용 공식 문서 경로로 사용할 수 있음

## 사용자 플로우

1. 사용자가 Home에 진입한다.
2. Hero에서 Yesica의 방향성과 주요 작업 톤을 파악한다.
3. Selected Projects에서 대표 작업과 제품 경험을 먼저 확인한다.
4. About과 Experience에서 실제 경력과 운영 경험을 확인한다.
5. 프로젝트 상세 페이지에서 문제, 목표, 과정, 솔루션 영역을 본다.
6. Contact로 이동해 협업 가능 범위와 문의 방식을 확인한다.

## 저장/동기화 방식

- 사이트 콘텐츠는 코드와 Markdown/TypeScript 데이터로 관리한다.
- 프로젝트 데이터는 `src/data/projects.ts`에서 관리한다.
- 문서는 `docs/` 하위 Markdown 파일로 관리한다.
- 배포는 GitHub 브랜치와 Vercel 프로젝트 연결을 사용한다.

## 예외/제한사항

- 이메일, 브런치, GitHub, LinkedIn 링크는 공개 전 최종 확인이 필요하다.
- 프로젝트 상세 콘텐츠는 1차 포트폴리오용 요약 문장으로 구성되어 있으며, 정식 케이스 스터디 작성이 필요하다.
- mobimobi는 주요 프로젝트 데이터와 Home Hero에서 제외하고, 향후 Lab 또는 Side Projects 영역 후보로 둔다.
- Writing/Brunch 페이지는 소개 문구만 있는 1차 상태다.
- Schedule Block 공식 페이지는 기존 정적 페이지 내용 기준이며, 앱 업데이트 시 정책 문구도 함께 갱신해야 한다.
- 디자인 이미지는 실제 화면 이미지 연결 전까지 gradient/mockup 형태의 프리뷰로 표현한다.

## 아직 구현하지 않은 기능

- 실제 프로젝트 이미지/영상 연결
- 프로젝트별 정식 케이스 스터디 작성
- Live Product / App Store / GitHub 링크 필드 추가
- Brunch 글 목록 연결
- Resume 페이지
- Lab 또는 Studio 페이지
- SEO/OG 이미지 정리

## 검증 기준

- `npm run lint` 통과
- `npm run build` 통과
- Home, About, Projects, Project Detail, Contact, Writing 라우트 접근 가능
- Schedule Block 공식 페이지 라우트 접근 가능
- 데스크톱과 모바일에서 Hero 텍스트와 카드가 겹치지 않음
- `main` 브랜치 배포는 실서버에 반영
- `dev` 브랜치 배포는 개발서버에 반영
